# `Period Calculator` 项目产品需求文档 (PRD)

**项目定位**：隐私优先、极致丝滑、温馨专业的下一代女性经期管理工具。

**核心理念**：通过“无登录、本地存储”解决大厂隐私痛点；通过“极致加载速度”和“AI 内容建模”挑战老牌站点。

------

## 一、 项目概述

- **目标关键词**：`period calculator` (核心词), `irregular period calculator`, `ovulation calculator` (长尾词)。
- **目标用户**：寻找快速预测工具、关注隐私安全、有经期异常或备孕/避孕需求的全球女性用户。
- **核心优势**：
  - **零摩擦**：点开即用，无需注册登录。
  - **隐私安全**：数据 100% 留在浏览器本地。
  - **极致体验**：支持夜间模式，毫秒级响应，符合 2026 年审美。

------

## 二、目标用户与使用场景

1. **快速查询型**：在旅行或约会前，想知道下次日期的女性。
2. **隐私敏感型**：不愿将生理数据上传到大厂 App 云端的女性。
3. **异常焦虑型**：经期不规律（Irregular），寻找比简单平均值更准的算法。
4. **跨端查看型**：在移动端计算，并希望快速存入手机日历。

------

## 三、 技术架构需求

- **框架**：Next.js (App Router)。
- **样式**：Tailwind CSS。
- **部署**：Vercel (利用 Serverless 和 Edge Network)。
- **数据存储**：
  - **用户侧**：LocalStorage / IndexedDB (禁止上传服务器)。
  - **内容侧**：MDX 或无头 CMS 用于存储长尾 FAQ。
- **Rendering**：首页文字内容使用 **SSG** (静态生成)，计算器组件使用 **Client Component**。
- **I18n**：使用 `next-intl`，路径设为 `/es`, `/fr`。
- **Themes**：使用 `next-themes` 处理深色模式，防止首屏闪烁。
- **Metadata**：在每个路由中使用 Next.js Metadata API 动态生成 TDK。

------

## 四、 功能需求模块

### 1. 核心计算器功能 (The Tool)

- **基础模式**：输入“上次月经首日” + “平均周期长度”。
- **进阶模式 (Irregular Mode)**：支持输入过去 3-6 个月的历史数据，算法通过**移动平均值和标准差**给出“预测窗口”而非单一日期。
- **结果可视化**：渲染动态日历组件，用不同颜色标注：
  - 红色：月经期 (Menstruation)
  - 蓝色：排卵/易孕期 (Fertile Window)
  - 黄色：经前期 (PMS)

### 2. “添加到日历”功能 (Add to Calendar)

- **逻辑**：前端生成 `.ics` 文件（Apple/Outlook）或 Google Calendar 跳转链接。
- **内容**：包含标题、预测时间段、以及返回网站的链接。

### 3. 主题与多语言切换

- **夜间模式**：支持手动切换及随系统自动切换。
- **语言切换**：英语为默认语言。首页提供语种选择器，切换时保持当前计算结果不丢失。

### 4. 移动端专项支持 (Mobile-First)

- **PWA (Progressive Web App)**：支持“添加到主屏幕”，显示独立 Logo，离线可访问计算器。
- **底部导航栏**：针对移动端单手操作，常用功能（计算器、不规律专题、多语言）放在底部。
- **胖手指友好**：所有交互按钮高度不低于 `48px`，间距不低于 `12px`。
- **手势操作**：支持日历滑动切换月份。

### 5. PWA 支持

- 提供“添加到主屏幕”功能，生成离线可用的 Web App。

## 五、 页面结构与内容策略 (Silo Structure)

### 1. 首页 (Pillar Page)，做主词`Period Calculator`

- **首屏**：核心计算器 + 隐私声明。
- **二屏**：计算原理说明（ How to calculate 步骤）。
- **三屏**：深度科普（为什么月经推迟、不规律周期的意义）。
- **底屏**：FAQ 模块（针对 3.6k 问题库的高频项）。

### 2. 子页面 (Cluster Pages)

- `/irregular-period-calculator/`：专门针对不规律人群。
- `/ovulation-calculator/`：针对备孕人群。
- `/blog/[slug]/`：动态路由，承接数千个长尾问题页面。

------

## 六、 SEO 与 E-E-A-T 规范

### 1. 结构化数据 (Schema)

- **WebApplication**：定义工具属性。
- **FAQPage**：让搜索结果出现问答下拉框。
- **HowTo**：详细说明计算步骤。

### 2. 页面元标签 (Meta Tags)

- **Title**：必须包含核心词，且根据语种自动本地化。
- **Canonical**：严格指定规范网址，防止多语言/参数导致的内容重复。
- **H1-H3**：首页 H1 必须包含 `period calculator`，H2 必须包含高频问题。

### 3. E-E-A-T 落地细节

- **Editorial Policy 页面**：说明公式来源（ACOG/NHS）。
- **References**：每篇文章文末必须有权威医疗外链引用。

------

## 七、 UI/UX 设计指南

- **风格**：温馨、柔和、专业（Morandi 莫兰迪色系）。
- **交互**：按钮使用大圆角（`rounded-2xl`），结果显示采用平滑渐入动画。
- **加载占位图 (Skeleton)**：在 Next.js 加载时显示骨架屏，消除视觉跳动（优化 CLS）。

------

## 八、 隐私与合规

- **GDPR/CCPA**：由于不存储数据到服务器，仅需简单的 Cookie 告知，但要在 `Privacy Policy` 中详述本地存储机制。
